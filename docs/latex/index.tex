{\bfseries{\doxylink{namespace_g_o_a_t}{GOAT}}} is a modular, scene-\/based C++ library for flexible simulation of electric fields and intensity distributions using geometrical optics, including explicit phase calculation for the simulation of interference phenomena.\hypertarget{index_autotoc_md1}{}\doxysection{\texorpdfstring{Why GOAT?}{Why GOAT?}}\label{index_autotoc_md1}
In optical modeling, each new problem often requires starting from scratch. {\bfseries{\doxylink{namespace_g_o_a_t}{GOAT}}} solves this by providing a reusable, extensible library designed for a wide range of optical tasks. Its architecture is scene-\/based and modular, allowing you to combine and extend components as needed.\hypertarget{index_autotoc_md2}{}\doxysection{\texorpdfstring{Key Features}{Key Features}}\label{index_autotoc_md2}

\begin{DoxyItemize}
\item {\bfseries{Abstract, extensible ray tracer\+:}} ~\newline
 Easily adapt the ray tracing engine to your problem.
\item {\bfseries{Scene architecture\+:}} ~\newline
 Define a simulation by specifying light sources and objects in a scene.
\item {\bfseries{Flexible interfaces\+:}} ~\newline
 Extend base classes for your own object and light source types.
\item {\bfseries{Ready-\/made sources and shapes\+:}} ~\newline
 Gaussian beams, top-\/hat sources, ellipsoids, and import of binary STL shapes.
\item {\bfseries{Efficient 3D math classes\+:}} ~\newline
 Matrix and Vector templates for mathematical operations (in {\ttfamily /maths}).
\item {\bfseries{Parallel processing for large STL files\+:}} ~\newline
 The {\ttfamily goat\+\_\+raytracing\+\_\+mp.\+lib} library offers {\bfseries{Open\+MP support}} to accelerate the loading and processing of large STL files.
\end{DoxyItemize}\hypertarget{index_autotoc_md3}{}\doxysection{\texorpdfstring{Library Structure}{Library Structure}}\label{index_autotoc_md3}

\begin{DoxyItemize}
\item {\bfseries{goat\+\_\+maths.\+lib\+:}} ~\newline
 Mathematical basics\+: 3D vector and matrix classes with operators.
\item {\bfseries{goat\+\_\+raytracing.\+lib\+:}} ~\newline
 Ray tracing engine, light sources, object representations.
\item {\bfseries{goat\+\_\+raytracing\+\_\+mp.\+lib\+:}} ~\newline
 Same as {\ttfamily goat\+\_\+raytracing.\+lib}, but with {\bfseries{Open\+MP parallelization}} for efficient loading and handling of large STL geometries.

{\bfseries{Requirements\+:}} ~\newline

\begin{DoxyItemize}
\item \href{https://cmake.org/}{\texttt{CMake}} (recommended)
\item C++17 compatible compiler (tested\+: Visual Studio 2019+, GCC)
\item For {\ttfamily goat\+\_\+raytracing\+\_\+mp.\+lib}\+: Open\+MP support enabled in your compiler.
\end{DoxyItemize}

{\bfseries{Build steps\+:}} ~\newline

\begin{DoxyEnumerate}
\item Open a terminal / console.
\item Navigate to your \doxylink{namespace_g_o_a_t}{GOAT} directory.
\item Run\+: cmake . {\itshape (Don’t forget the dot!)}
\end{DoxyEnumerate}

CMake detects your system and generates project files or a Makefile.
\begin{DoxyItemize}
\item {\bfseries{Windows (VS 2019+)\+:}} Open the \doxylink{namespace_g_o_a_t}{GOAT} directory directly with Visual Studio, or use CMake as above.
\item {\bfseries{Linux\+:}} Run {\ttfamily make} after {\ttfamily cmake}.
\end{DoxyItemize}
